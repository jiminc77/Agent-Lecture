# Task 2. 비명시적인 내 역할 찾기

## 이번 태스크에서 배우는 것

한 문장에 내 일이 완성된 형태로 쓰이지 않아도, 여러 발화의 맥락을 묶어 **내가 해야 할 일정 후보**를 찾습니다.

이번에는 다음 2가지를 작성합니다.

- `Personal relevance rules`
- `Exclusion rules`

## 상황 보기

![Task 2 입력 화면](../../../task%202.png)

실제 opencode agent는 같은 내용을 담은 `raw-text.md` 파일을 읽습니다.

## 목표 결과

아래 정보가 output 폴더의 intake.md에 출력되어야 합니다.

### 일정 후보

#### 고객사 B 자료 전달

| 항목 | 내용 |
|---|---|
| 제목 | 고객사 B 자료 전달 |
| 카테고리 | follow_up |
| 원문 근거 | "최종본은 현우님 쪽에서 들고 계시죠?" / "현우: 네, 제가 확인해볼게요." / "그럼 내일 오전 10시까지 고객사에 전달되면 좋겠습니다." |
| 날짜/시간 표현 | 2026-05-23 10:00 |
| 확인 필요 | 없음 |

### 제외한 항목

| 원문 | 제외 이유 |
|---|---|
| "수진: 저는 계약서 문구만 따로 볼게요." | 수진님에게 배정된 일 |
| "디자인 시안은 유진님이 월요일까지 업데이트하기로 했습니다." | 유진님에게 배정된 일 |

## 목표 체크

- 고객사 B 자료 전달이 일정 후보로 포함된다.
- 최종본을 현우가 들고 있다는 말과 민수의 전달 시한을 함께 묶어 판단한다.
- 앞의 "오늘 안"은 고객사의 희망이고, 뒤의 "내일 오전 10시까지"를 합의된 전달 시한으로 본다.
- 날짜/시간이 `2026-05-23 10:00`으로 정리된다.
- 수진님의 계약서 문구 검토는 제외된다.
- 유진님의 디자인 시안 업데이트는 제외된다.
