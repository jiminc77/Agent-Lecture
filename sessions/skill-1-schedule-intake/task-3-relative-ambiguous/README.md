# Task 3. 상대 날짜와 모호한 표현 다루기

## 이번 태스크에서 배우는 것

`다음주 이 시간`, `오전 중`, `후속 회의 전` 같은 표현을 무리하게 확정하지 않고, 확정 가능한 값과 확인이 필요한 값을 분리합니다.

이번에는 다음 2가지를 작성합니다.

- `Date and ambiguity handling`
- `Procedure` 보강

## 상황 보기

아래 이미지는 실습 입력을 메모 화면처럼 표현한 것입니다.

![Task 3 입력 화면](../../../task%203.png)

실제 opencode agent는 같은 내용을 담은 `raw-text.md` 파일을 읽습니다.

## 오른쪽 창에서 수정할 파일

```text
SKILL.md
```

Task 3의 `SKILL.md`에는 Task 1-2 정답이 이미 들어 있습니다. 이번 태스크에서는 날짜와 모호성 처리 규칙만 채우면 됩니다.

작성 후 현재 태스크 폴더에서 아래 명령을 실행합니다.

```bash
./apply-skill.sh
```

## 실행 방법

현재 태스크 폴더에서 opencode를 실행하고 아래처럼 요청합니다.

```text
raw-text.md를 읽고, 상대 날짜와 모호한 표현을 구분해서 일정 후보를 정리해줘.
```

## 목표 체크

- 후속 회의가 일정 후보로 포함된다.
- `다음주 이 시간`은 기준 시각을 사용해 `2026-05-29 14:00`으로 해석된다.
- 고객 피드백 정리는 `후속 회의 전` 준비 일정으로 남는다.
- 재무팀 비용 추정치 공유는 `2026-05-27 오전`까지는 해석하되 정확한 시간은 확인 필요로 남긴다.
- 유진님의 디자인 시안 업데이트는 제외된다.
- `오전 중`, `후속 회의 전` 같은 표현을 임의의 정확한 시각으로 바꾸지 않는다.

## 참고: 기대 출력 형태

```markdown
## 일정 후보

| id | 제목 | 카테고리 | 원문 근거 | 날짜/시간 표현 | 확인 필요 |
|---|---|---|---|---|---|
| C1 | 후속 회의 | meeting | "다음주 이 시간 같은 멤버로 후속 회의" | 2026-05-29 14:00 | 없음 |
| C2 | 고객 피드백 정리 | preparation | "후속 회의 전까지 고객 피드백 정리하기" | 후속 회의 전 | 정확한 시간 확인 필요 |
| C3 | 재무팀 비용 추정치 공유 | follow_up | "수요일 오전 중에 재무팀에 비용 추정치 공유하기" | 2026-05-27 오전 | 오전 몇 시인지 확인 필요 |

## 제외한 항목

| 원문 | 제외 이유 |
|---|---|
| "디자인 시안은 유진님이 업데이트" | 유진님에게 배정된 일 |
```
