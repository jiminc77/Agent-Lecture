# Task 6. 최종 확인과 오류 방지

## 이번 태스크에서 배우는 것

여러 이벤트를 만들 때 중복과 시간 오류를 확인하고, 최종 파일을 쓰기 전에 사용자 확인을 받습니다.

이번에는 다음 4가지를 작성합니다.

- `Confirmation policy`
- `Duplicate and time checks`
- `Validation checklist` 보강
- `Procedure` 보강

## 이번 단계의 입력

실제 opencode agent는 `intake.md` 파일을 읽습니다. 아래는 그 안에서 이번 태스크에 필요한 일정 후보 부분입니다.

## 목표 결과

바로 `schedule-preview.md`를 확정하지 않고, 먼저 아래처럼 확인해야 합니다.

```text
캘린더 이벤트를 만들기 전에 확인이 필요합니다.

- "후속 회의"와 "후속 회의 리마인드"는 같은 원문과 같은 시각에서 나온 중복 후보로 보입니다. 하나의 일정으로 합칠까요?
- "후속 회의"는 시작 시각만 있습니다. 소요 시간은 얼마로 둘까요?
- "고객 피드백 정리"는 후속 회의 전 준비 일정이지만 정확한 시작 시간과 소요 시간이 없습니다. 언제, 얼마 동안 넣을까요?
- "재무팀 비용 추정치 공유"는 2026-05-27 오전으로만 해석됩니다. 몇 시로 넣고, 소요 시간은 얼마로 둘까요?
```

## 목표 체크

- "후속 회의"와 "후속 회의 리마인드"가 중복일 수 있음을 먼저 확인한다.
- 두 후보를 확인 없이 별도 이벤트 2개로 만들지 않는다.
- "후속 회의"의 소요 시간, "고객 피드백 정리"의 정확한 시간과 소요 시간, "재무팀 비용 추정치 공유"의 정확한 시간과 소요 시간을 질문한다.
- 종료 시각이 시작 시각보다 늦은지 확인한다.
- 준비 일정은 준비 대상 회의보다 앞에 있어야 한다.
- 최종 JSON 파일을 쓰기 전에 사용자 확인을 받는다.


## 실행 방법

현재 태스크 폴더에서 opencode를 실행하고 아래처럼 요청합니다.

```text
intake.md를 읽고, 캘린더 이벤트 JSON을 만들기 전에 중복과 시간 오류를 검토해줘.
문제가 있으면 먼저 확인해줘.
```

opencode agent가 확인 질문을 하면 같은 대화창에 바로 답합니다. 예를 들어 아래처럼 답할 수 있습니다.

```text
"후속 회의"와 "후속 회의 리마인드"는 하나로 합치고, 후속 회의는 1시간으로 해줘.
고객 피드백 정리는 2026-05-29 13:00에 30분,
재무팀 비용 추정치 공유는 2026-05-27 10:00에 30분으로 해줘.
```

답변 뒤 agent가 계속 진행하지 않으면 아래처럼 이어서 요청합니다.

```text
방금 답변을 반영해서 schedule-preview.md를 만들어줘.
```
