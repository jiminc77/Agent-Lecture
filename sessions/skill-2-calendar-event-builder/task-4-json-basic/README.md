# Task 4. 일정 후보를 JSON 이벤트로 변환하기

## 이번 태스크에서 배우는 것

`intake.md`에 정리된 일정 후보를 읽고, 캘린더에 넣을 수 있는 JSON 이벤트로 바꾸는 `calendar-event-builder` Skill의 기본 골격을 만듭니다.

이번에는 다음 2가지만 작성합니다.

- `When to use`
- `Procedure`

## 이번 단계의 입력

실제 opencode agent는 `intake.md` 파일을 읽습니다. 아래는 그 안에서 이번 태스크에 필요한 일정 후보 부분입니다.

이번 입력은 JSON 변환 연습에 집중할 수 있도록 고객사 A 견적 콜의 종료 시각을 11:00까지 정리해 둔 실습용 입력입니다.

### 일정 후보

#### 고객사 A 견적 콜

| 항목 | 내용 |
|---|---|
| 제목 | 고객사 A 견적 콜 |
| 카테고리 | meeting |
| 원문 근거 | "현우님은 내일 오전 10시에 고객사 A와 견적 관련 콜을 진행합니다." |
| 날짜/시간 표현 | 2026-05-23 10:00-11:00 |
| 확인 필요 | 없음 |

### 제외한 항목

| 원문 | 제외 이유 |
|---|---|
| "회의 후 견적서 초안은 민수님이 작성합니다." | 민수님에게 배정된 일 |
| "수진님은 회의록을 정리해서 채널에 올려주세요." | 수진님에게 배정된 일 |

## 목표 결과

`output/schedule-preview.md`에 아래 JSON 코드블록이 들어가야 합니다.

```json
{
  "title": "고객사 A 견적 콜",
  "start": "2026-05-23T10:00:00",
  "end": "2026-05-23T11:00:00",
  "notes": "카테고리: meeting. 원문 근거: 현우님은 내일 오전 10시에 고객사 A와 견적 관련 콜을 진행합니다."
}
```

## 목표 체크

- 고객사 A 견적 콜이 JSON 이벤트로 생성된다.
- `title`, `start`, `end`, `notes` 필드만 사용한다.
- `start`는 `2026-05-23T10:00:00`이다.
- `end`는 `2026-05-23T11:00:00`이다.
- `end`는 `start`보다 늦다.

SKILL.md
