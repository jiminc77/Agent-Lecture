# Task 5. 부족한 값 질문하기

## 이번 태스크에서 배우는 것

캘린더 이벤트 JSON을 만들 수 있는지 필수 값과 형식을 먼저 검증합니다. 필요한 값이 부족하면 바로 파일을 쓰지 않고 사용자에게 짧고 구체적으로 질문합니다.

이번에는 다음 3가지를 작성합니다.

- `Validation checklist`
- `Clarification policy`
- `Procedure` 보강

## 이번 단계의 입력

실제 opencode agent는 `intake.md` 파일을 읽습니다. 아래는 그 안에서 이번 태스크에 필요한 일정 후보 부분입니다.

이 입력은 검증과 질문을 연습하기 위해 일부러 시간이 모호하게 만든 별도 실습용 입력입니다. Task 2의 결과와 같은 사건처럼 보이더라도, 이번 태스크에서는 아래 표를 기준으로 판단합니다.

## 일정 후보

| id | 제목 | 카테고리 | 원문 근거 | 날짜/시간 표현 | 확인 필요 |
|---|---|---|---|---|---|
| C1 | 고객사 B 자료 전달 | follow_up | "최종본은 현우님 쪽에서 들고 계시죠?" / "현우: 네, 제가 확인해볼게요." / "그럼 내일 오전까지 고객사에 전달되면 좋겠습니다." | 2026-05-23 오전 | 오전 몇 시인지 확인 필요 |

## 제외한 항목

| 원문 | 제외 이유 |
|---|---|
| "수진: 저는 계약서 문구만 따로 볼게요." | 수진님에게 배정된 일 |
| "디자인 시안은 유진님이 월요일까지 업데이트" | 유진님에게 배정된 일 |

## 오른쪽 창에서 수정할 파일

```text
SKILL.md
```

Task 5의 `SKILL.md`에는 Task 4 정답이 이미 들어 있습니다. 이번 태스크에서는 검증과 질문 규칙만 채우면 됩니다.

작성 후 현재 태스크 폴더에서 아래 명령을 실행합니다.

```bash
./apply-skill.sh
```

## 실행 방법

현재 태스크 폴더에서 opencode를 실행하고 아래처럼 요청합니다.

```text
intake.md를 읽고, 캘린더 이벤트 JSON을 만들 수 있는지 검증해줘.
부족한 값이 있으면 먼저 질문해줘.
```

opencode agent가 확인 질문을 하면 같은 대화창에 바로 답합니다. 예를 들어 아래처럼 답할 수 있습니다.

```text
오전 10시, 30분으로 해줘.
```

답변 뒤 agent가 계속 진행하지 않으면 아래처럼 이어서 요청합니다.

```text
방금 답변을 반영해서 schedule-preview.md를 만들어줘.
```

## 목표 체크

- `2026-05-23 오전`을 임의로 정확한 시각으로 바꾸지 않는다.
- JSON을 바로 쓰기 전에 정확한 시간과 소요 시간을 질문한다.
- 제외 항목은 캘린더 이벤트로 만들지 않는다.
- 사용자가 예를 들어 `오전 10시, 30분`이라고 답한 뒤에만 `schedule-preview.md`를 만든다.

## 참고: 목표 질문 예시

```text
캘린더 이벤트를 만들기 전에 확인이 필요합니다.

1. "고객사 B 자료 전달"은 2026-05-23 오전 중 몇 시로 넣을까요?
2. 이 일정의 소요 시간은 얼마로 둘까요?
```

## 참고: 확인 뒤 기대 JSON

사용자가 예를 들어 `오전 10시, 30분`이라고 답하면 `output/schedule-preview.md`에 아래와 같은 JSON이 포함되어야 합니다.

```json
{
  "title": "고객사 B 자료 전달",
  "start": "2026-05-23T10:00:00",
  "end": "2026-05-23T10:30:00",
  "notes": "카테고리: follow_up. 원문 근거: 최종본은 현우님 쪽에서 들고 계시죠? / 현우: 네, 제가 확인해볼게요. / 그럼 내일 오전까지 고객사에 전달되면 좋겠습니다."
}
```
